Writes content to a file in the local filesystem.

Parameters:
- filePath (required): Path to the file to write. Use RELATIVE paths (e.g., "add.py", "src/utils/helper.py"). NEVER use absolute paths starting with "/".
- content (required): The content to write to the file.
- create (optional): Set to true to create parent directories if needed.

Usage:
- This tool will overwrite an existing file if there is one at the provided path.
- If this is an existing file, you MUST use the Read tool first to read the file's contents. This tool will fail if you did not read the file first.
- If the target file is not explicitly specified by the user, run Glob first to inspect the repository and prefer adding code to existing relevant files over creating a new file.
- ALWAYS prefer editing existing files in the codebase. NEVER write new files unless explicitly required.
- NEVER proactively create documentation files (*.md) or README files. Only create documentation files if explicitly requested by the User.
- Only use emojis if the user explicitly requests it. Avoid writing emojis to files unless asked.

Example:
{
  "filePath": "add.py",
  "content": "def add(a, b):\n    return a + b\n"
}
